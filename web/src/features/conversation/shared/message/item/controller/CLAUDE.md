# 消息项控制层

- `use-message-item-projection.ts` 负责把原始轮次消息归一化为可渲染投影，不持有视觉交互状态。
- `use-message-item-controller.ts` 只组合展开、复制、停止和流式布局等交互状态，并向视图返回具体模型。
- 排序、权限匹配、结果统计和最终回复选择保持为纯函数；不要在视图组件中复制这些规则。
- 隐藏工具使用集合匹配，避免在消息块循环中反复线性查找。
- 活动状态与过程摘要由 `message-item-activity.ts` 的规则表统一推导；运行阶段、流式块和待确认状态按显式优先级合并。
- 消息排序固定经过内容规则投影、系统事件分区、工具关联和源顺序归并；匹配工具的事件紧随工具，同源无匹配事件保持后置。
