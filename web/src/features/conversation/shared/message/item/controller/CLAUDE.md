# 消息项控制层

- `use-message-item-controller.ts` 组合显示推导、展开、复制、停止和流式布局，并返回按 User/Assistant 与视觉职责分组的具体模型。
- `projection/` 负责内容排序、系统事件、权限、统计和最终回复等纯投影，不持有视觉交互状态。
- `display/` 分离纯显示状态机和过程展开生命周期，只消费投影 Hook 的最小结构契约。
- 控制器返回值保持类型推断；消费视图自行声明窄契约，禁止建立覆盖所有视图的中心状态接口。
- 排序、权限匹配、结果统计和最终回复选择保持在 `projection/` 的纯函数中；不要在视图组件中复制这些规则。
- 隐藏工具使用集合匹配，避免在消息块循环中反复线性查找。
- 结果文案与统计字段由 `message-item-stats.ts` 的有序规则和独立字段投影生成；聚合函数不得重复解释 subtype、数值区间或 usage。
- 控制器只组合相邻 `activity/` 与 `process/` 的纯领域结果，不得重新推导活动优先级、过程摘要或问答超时。
- 系统消息 subtype 的展示信息由 `projection/message-item-system-events.ts` 的映射表定义，不得回流到协议类型或恢复条件链。
- 消息排序固定经过内容规则投影、系统事件分区、工具关联和源顺序归并；匹配工具的事件紧随工具，同源无匹配事件保持后置。
