# 定时任务

## 目录边界

- 根目录只保留页面装配、概览、实时刷新和跨子域格式化。
- `controller/` 负责列表资源、写命令、并发状态和反馈。
- `history/` 负责运行记录资源、命令、弹窗壳层和分层视图。
- `list/` 负责列表状态、单项展示、排序和展示投影。
- `dialog/` 负责创建与编辑表单，`pickers/` 负责时间选择。

## 约定

- 页面只加载定时任务资源，不附带 Heartbeat 或其他自动化资源。
- 写命令先提交 API 返回的权威对象，再静默刷新做服务端对账。
- Agent 通知使用任务自身的 `agent_id`，不得使用页面默认 Agent 代替。
- 运行历史请求和动作必须绑定当前 Job；切换或关闭弹窗后，旧结果不得写回。
