# 技能市场

## 目录边界

- 根目录只保留页面编排、Header、搜索入口和导览定义。
- `controller/` 管理目录查询、外部搜索、来源配置、异步操作与反馈状态。
- `catalog/` 负责技能目录和更新提示；来源、安装状态与更新检查统一由 `skills-catalog-model.ts` 投影。
- `external/` 负责社区来源结果、来源管理和预览；身份、来源、导入状态与预览内容统一由纯模型投影。
- `import/` 负责本地与 Git 导入表单。
- `detail/` 按路由编排、请求控制器、展示模型和纯视图分离技能详情；Markdown 与部署失败说明留在详情域。

## 约定

- 子视图只接收自身消费的小接口，禁止传入完整市场控制器。
- 目录与详情不得根据 `source_type` 维护平行兜底；新增来源必须先补齐各自的穷尽映射。
- 外部技能身份与导入状态只允许由 `external-skill-model.ts` 生成，controller 不持有展示投影。
- 异步请求的竞态由所属 controller 处理，视图不得自行维护请求版本或修正结果。
- 详情更新与删除复用市场命令，命令用明确结果决定后续刷新和导航，不重复调用 API。
- 新增流程时先确定状态归属，再决定目录；不得把领域逻辑放回通用 `hooks/`。
